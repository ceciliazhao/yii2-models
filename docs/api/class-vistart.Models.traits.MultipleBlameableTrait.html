Trait MultipleBlameableTrait

一个模型的某个属性可能对应多个责任者，该 trait 用于处理此种情况。此种情况违反 了关系型数据库第一范式，因此此 trait 只适用于责任者属性修改不频繁的场景，在开 发时必须严格测试数据一致性，并同时考量性能。 Basic Principles:

  1. when adding blame, it will check whether each of blames including to be added is valid.
  2. when removing blame, as well as counting, getting or setting list of them, it will also check whether each of blames is valid.
  3. By default, once blame was deleted, the guid of it is not removed from list of blames immediately. It will check blame if valid when adding, removing, counting, getting and setting it. You can define a blame model and attach it events triggered when inserting, updating and deleting a blame, then disable checking the validity of blames.

Notice:

  1. You must specify two properties: $multiBlamesClass and $multiBlamesAttribute.
    • $multiBlamesClass specify the class name of blame.
    • $multiBlamesAttribute specify the field name of blames.
  2. You should rename the following methods to be needed optionally.
